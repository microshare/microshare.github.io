End-to-End Flow Overview

A sensor fires an event. The bundler resolves configuration, matches a crew roster, and creates or updates a ticket.

IoT Event Sensor fires (e.g. soap 80% empty)
Bundler Client-side JS orchestrator
Config API PATCH /config endpoint
Config Processor 12-layer merge + roster match
Decision Create, update, or resolve?
Workflow Engine Ticket created or updated

System Components Architecture

Nashorn Bundler

JavaScript (Nashorn engine) · bundler.js

Client-side orchestrator on the stream server. Receives IoT events, fetches config, and decides the ticket action.

ConfigResource

Scala (Pekko HTTP) · ConfigResource.scala

HTTP route exposing PATCH /config/{recType} for configuration evaluation requests.

ConfigProcessor

Scala · ConfigProcessor.scala

Server-side engine that resolves the 12-layer hierarchical config and matches roster records.

Workflow Engine

Internal service · Workflow API

Creates, updates, and resolves workflow tickets/incidents. Receives the assembled ticket payload.

Configuration Hierarchy Config

A single config record contains settings at four levels of specificity. More specific levels override general ones.

data
  config/ ← Base defaults (all events)
    ticket: { threshold, priority, timing, escalation, tasks, … }
  solutions/
    {solution}/ ← e.g. "clean"
      config/ ← Solution-level overrides
      alerts/
        {alert}/ ← e.g. "soap", "trash"
          config/ ← Alert-level overrides
          events/
            {event}/ ← e.g. "fill_50", "active"
              config/ ← Most specific override
  locations[] ← Per-device / per-area overrides (matched by longest prefix)

The 12-Layer Deep Merge Core Logic

Three config sources × four specificity levels = 12 layers. Later layers override earlier ones. This is how a single sensor event gets its final, tailored configuration.

Default Record (global/shared)
1.config
2.solutions.{solution}.config
3.solutions.{solution}.alerts.{alert}.config
4.solutions.{solution}.alerts.{alert}.events.{event}.config
User Record (tenant-specific)
5.config
6.solutions.{solution}.config
7.solutions.{solution}.alerts.{alert}.config
8.solutions.{solution}.alerts.{alert}.events.{event}.config
Location Match (best-match device path)
9.config
10.solutions.{solution}.config
11.solutions.{solution}.alerts.{alert}.config
12.solutions.{solution}.alerts.{alert}.events.{event}.config
Objects → merged recursively Arrays → concatenated Scalars → last layer wins
Location matching uses longest-prefix match on the device tag path (e.g. ["A","0","241","AS"]).

Roster (Route) Assignment Routing

Rosters determine which work crew or tour receives a ticket. The server evaluates every active roster rule against the incoming event.

Incoming Event
Solution: clean
Alert: incident
Event: high_threshold
Location: PT → 0 → 442A → LS
MATCH
Matched Roster
Name: Tour 417
Covers: PT/0/442A, PT/0/444A
Events: medium_threshold, high_threshold
Result: roster ID + name → ticket

Solution

Must match event solution. "*" = any.

Alert

String or array match. "*" = any.

Events

Array of event names. "*" = any.

Location

Nested tree walk with "*" wildcards at each level.

Bundler Decision Logic Decision Tree

After configuration is resolved, the bundler decides what to do with the event.

Check Is this event set to "avoid" or type "good"?
Yes → Skip event entirely
No → Are solution, alert, AND event all present?
No → Log error, return
Yes → Search for open tickets at this location
Open ticket found
autoResolve? Send auto-resolve signal to workflow
staffResolve? Send staff-resolve signal to workflow
Otherwise Add this event as a source to the existing ticket
No open ticket
Resolve-only event? Nothing to resolve, skip
Otherwise → Create new ticket via Workflow API

deviceTag: Aggregation Granularity Key Concept

The deviceTag setting controls how broadly events are grouped into the same ticket.

deviceTag = "none" (default)
Location: A / 0 / 241 / AS
Matches only: A / 0 / 241 / AS

Full precision. Each specific device gets its own ticket.

deviceTag = "3"
Location: A / 0 / 241 / AS
Matches: A / 0 / 241 / *

Truncated to 3 tags. Events from sub-locations merge into one ticket per area.
